## How it works

### Workspace dependencies

The root `package.json` uses npm/pnpm/yarn workspaces to link packages together. Each package under `packages/` can be imported by the app using its workspace name.

```json
{
  "workspaces": ["packages/*"]
}
```

After adding the `workspaces` field to your root `package.json`, run your package manager's install command to link everything:

```bash
npm install
```

### Using shared packages from the app

The `@project/shared-ui` package exposes components that the app can import directly:

```tsx
import { Button } from "@project/shared-ui";
```

Turborepo detects the dependency and builds `shared-ui` before the app when you run `turbo build`.

### Turborepo pipeline

The `turbo.json` file defines how tasks (build, dev, lint) run across the workspace:

- `build` — runs in dependency order (`^build`), caches `.next/` and `dist/` outputs
- `dev` — runs in parallel with no caching
- `lint` — runs in dependency order

Run `npx turbo build` to build all packages and the app.

## Adding a new package

1. Create a directory under `packages/`:

```bash
mkdir packages/my-package
```

2. Add a `package.json`:

```json
{
  "name": "@project/my-package",
  "version": "0.1.0",
  "private": true,
  "exports": "./src/index.ts",
  "devDependencies": {
    "typescript": "^5.0.0"
  }
}
```

3. Create source files:

```bash
mkdir packages/my-package/src
echo "export const greeting = 'hello';" > packages/my-package/src/index.ts
```

4. Import it from the app or another package:

```ts
import { greeting } from "@project/my-package";
```

## Customization guide

### Renaming packages

Replace `@project/` prefix with your own scope:

- Update `name` in each `packages/*/package.json`
- Update `exports` and `main` fields if needed
- Update cross-package references (e.g., `tsconfig.json` `extends` paths)
- Update the `.eslintrc.js` root config to match

### Adding a new shared config package

Copy the pattern from `config-ts` or `config-eslint`:

```bash
mkdir packages/config-my-tool
```

Add `package.json` with a unique name like `@project/config-my-tool`, then reference it from the app's config files using `extends`.

### Adding more apps

To add a second app (e.g., an admin dashboard):

1. Create `apps/admin/` directory
2. Scaffold a new Next.js app inside it
3. Reference shared packages: `"@project/shared-ui": "workspace:*"`
4. Add `apps/admin/` to `turbo.json` tasks if needed

The existing packages are already set up with TypeScript path aliases that support the `apps/*` structure.

### TypeScript path aliases

The app's `tsconfig.json` uses `@/` to reference `src/`. Shared packages are imported by package name (e.g., `@project/shared-ui`). If you move the app into `apps/web/`, update the root `tsconfig.json` to include path references:

```json
{
  "compilerOptions": {
    "paths": {
      "@/*": ["./apps/web/src/*"]
    }
  }
}
```
